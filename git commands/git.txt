1 git add добавляет изменение из рабочего каталога в раздел проиндексированных файлов
2 git log. Команда git log отображает отправленные снимки состояния и позволяет просматривать и фильтровать историю проекта, а также искать в ней конкретные изменения. С помощью git status можно просматривать рабочий каталог и раздел проиндексированных файлов, в то время как git log показывает только историю коммитов
3 git commit делает для проекта снимок текущего состояния изменений, добавленных в раздел проиндексированных файлов.
4 git push — это консольная команда, которая передаёт в удалённый репозиторий изменения, сделанные в локальном репозитории
5 git merge объединяет несколько последовательностей коммитов в общую историю. Чаще всего команду git merge используют для объединения двух веток.
6 git rebase позволяющий объединить изменения двух веток. У этого способа есть преимущество перед merge (слияние) — он позволяет переписать историю ветки, придав тот истории тот вид, который нам нужен
7 git fetch — это основная команда, которая используется для загрузки содержимого из удаленного репозитория. Чтобы обновить локальный репозиторий до состояния удаленного репозитория
8 git pull используется для извлечения и загрузки содержимого из удаленного репозитория и немедленного обновления локального репозитория этим содержимым
